# SoulPod Project Core

## 1. 核心愿景 (Core Mission)
**见字如面，温润如初。** 本项目旨在通过“人格解耦”技术，为逝去的亲人构建一个永恒的数字载体。它不仅是对话工具，更是情感的延续与记忆的避风港。

## 2. 核心架构：三位一体 (The Trinity Architecture)
所有开发必须遵循以下标准件的解耦原则：
1. **长期记忆 (RAG Memory)**：基于原始聊天记录提取，以第一人称重构的事实库。
2. **人物画像 (User Profile)**：包含“五大人格(0-1)”参数、特定语言风格及亲属关系的结构化定义。
3. **行为规范 (System Prompts)**：定义“陪伴者”身份，设定情感边界（安慰为主，拒绝医疗/法律等非角色建议）。

## 3. 标准化资产规范 (DigitalTwinPackage)
一个合法的 SoulPod 载体必须包含以下目录结构：
- `/profile.json` (性格参数与口头禅)
- `/system_prompts.txt` (交互准则与身份设定)
- `/config.json` (技术底座配置：Embedding 模型、检索阈值)
- `/memories/` (持久化向量库与原始切片数据)

## 4. 核心准则 (The Golden Rules)
- **拒绝幻觉**：若记忆库无相关事实，宁可表达模糊印象，严禁编造关键人生经历。
- **去 AI 化**：禁止提及“模型”、“指令”或“算法”。在任何终端下，表现出的必须是那个“特定的人”。
- **隐私至上**：亲人的数据是神圣的资产，所有技术适配必须优先保护数据的本地化与私密性。

## 5. 开发演进路线
1. **数据炼金 (Extraction)**：将碎片化聊天记录清洗为具备第一人称情感密度的记忆块。
2. **灵魂建模 (Persona)**：基于素材自动化提取性格参数（Big Five）与语言模式。
3. **无缝陪伴 (Adaptation)**：通过 MCP 或私有协议，将 SoulPod 挂载至用户最常用的交互设备，且保持体验的一致性。
